1. Общие сведения
Программа является Windows приложением.

2. Необходимое окружение
Microsoft Windows XP и выше. Требует установленного .NET Framework 4.5

3. Описание работы приложения
Для запуска программы требуется файл лицензии "license.un1t"
Программа имеет 2 режима:
 a) интерактивное приложение - требуется для ввода настроек
 b) служба Windows - удобно для запуска программы на сервере
Для запуска режима интерактивного приложения необходимо запустить ярлык "Start config" либо файл "UN1TCounter.exe" с параметром /frm
Установить приложение как службу можно введя в Командной строке под учетной записью Администратора команду вида:
C:\Windows\Microsoft.NET\Framework\v4.0.30319\installutil.exe D:\...\UN1TCounter\Program\UN1TCounter.exe
Далее службу необходимо Запустить (Панель управления->дминитсрирование->Службы->UN1TCounter)

4. Сбор показаний с устройств
Для сбора показаний с устройств используется программа "snmpget.exe"
В файле лицензии указан список серийных номеров устройств для которых производится сбор показаний.
В настройках программы необходимо указать диапазоны ip адресов в которых находятся устройства.
Программа посылает сообщения (протокол SNMP) по ip адресам в указанных диапазонах, проверяет наличие серийного номера устройства в файле лицензии и сохраняет данные о счетчике устройства

5. Отправка данных на сервер ЮНИТ
В настройках программы необходимо указать настройки почтового сервера (SMTP), так же для сохранения отправленных сообщений в папке Отправленные (Sent) необходимо установить соответствующий флаг и ввести настройки IMAP
В соответствии с заданной периодичностью приложение производит отправку данных на сервер ЮНИТ посредством электронной почты.
После удачной отправки сообщения собранная ранее информация удаляется.

Отправляется следующая информация:
	Общие сведения - внутренний ID клиента, дата отправки, версия программы
	Список показаний устройств - серийный номер, дата запроса, текущий ip адрес, OID показателя общего счетчика, значение общего счетчика, тип показателя для внутренних нужд
	Формат (OID = тип: значение) - iso.3.6.1.4.1.367.3.2.1.2.1.4.0 = STRING: V8204300581		

Пример письма:
<DeviceRequestRoot>
<SysInfo idContractor="12131935" dateSend="02-02-2015 16:20:04 +05:00" progVersion="1.0" />
<DeviceRequestList>
<DeviceRequest serialNum="iso.3.6.1.4.1.367.3.2.1.2.1.4.0 = STRING: V8204300581 " date="02.02.2015 13:35:28" host="192.168.1.12" oid=".1.3.6.1.2.1.43.10.2.1.4.1.1" value="iso.3.6.1.2.1.43.10.2.1.4.1.1 = Counter32: 248782 " oidType="TotalCounter" />
</DeviceRequestList>
</DeviceRequestRoot>
